# Implementation Plan: Feature Flag Tracker

**Branch**: `012-feature-flag-tracker` | **Date**: 2026-03-04 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/012-feature-flag-tracker/spec.md`

## Summary

Extract frontend (TypeScript `FeatureFlags`) and backend (Kotlin `FeatureConfig`) feature flag values from each city's source code via GitHub API, persist as `data/feature-flags.json`, and display an interactive comparison matrix in a new "Ominaisuudet" tab. Regex-based parsing (no AST dependencies) since source file formats are well-known and stable.

## Technical Context

**Language/Version**: TypeScript 5.x on Node.js 20+ (data pipeline); vanilla JavaScript ES modules (frontend)
**Primary Dependencies**: axios (HTTP), existing GitHub API client (`src/api/github.ts`) — no new dependencies
**Storage**: JSON file (`data/feature-flags.json`) — new file alongside existing `data/current.json`
**Testing**: Jest + nock (unit/integration for parsers and API), Playwright (E2E for frontend)
**Target Platform**: GitHub Pages (static site) + GitHub Actions CI/CD
**Project Type**: Web application (data pipeline + static site)
**Performance Goals**: Feature flags view loads and displays full matrix within 2 seconds (SC-006)
**Constraints**: No framework, minimal dependencies, regex-based source code parsing, < 200ms parse time per file
**Scale/Scope**: 12 cities × ~35 frontend flags × ~15 backend config fields = ~600 cells in comparison matrix
